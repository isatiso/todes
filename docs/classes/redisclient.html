expire

  • expire(key: string, ttl: number): Promise<0 | 1>
    • Defined in redis-client.ts:177
    起始版本：1.0.0
时间复杂度：O(1)

    设置 key 的超时时间，到期后 key 会自动删除。

    当 删除 key 或 重写 值的时候，例如 DEL， SET，GETSET 和所有 *STORE 命令， 超时时间设置会被清除。 其他的诸如 INCR LPUSH 等 概念上修改 了值的操作，不会影响超时时间。

    返回值含义：

    • 1 ttl 设置成功。
    • 0 key 不存在，设置失败。

    Parameters

    • key: string
    • ttl: number

      需要设置的超时时间。

    Returns Promise<0 | 1>

    查看原始定义

keys

  • keys(pattern: string): Promise<string[]>
    • Defined in redis-client.ts:92
    起始版本：1.0.0
时间复杂度：O(N)，N 为当前库中 key 的个数。

    返回匹配 pattern 的全部 key 的列表。

    警告： 这是一个方便调试的命令，当你对一个庞大的库使用这个命令时，会导致性能及其底下。

    支持的 glob 风格 patterns:

    • h?llo 匹配 hello, hallo 和 hxllo。
    • h*llo 匹配 hllo 和 heeeello。
    • h[ae]llo 匹配 hello，hallo，但是 hillo 不行。
    • h[^e]llo 匹配 hallo， hbllo，但是 hello 不行。
    • h[a-b]llo 匹配 hallo 和 hbllo。

    Parameters

    • pattern: string

      glob 风格匹配模式。

    Returns Promise<string[]>

    查看原始定义
